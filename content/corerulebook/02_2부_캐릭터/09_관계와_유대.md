# 02-9. 관계와 유대

---

캐릭터는 혼자 살지 않습니다. 가족, 동료, 연인, 라이벌, 스승, 빚. 이 관계의 그물이 PC를 **이야기의 중심**에 두는 일입니다. FEWK는 이 그물을 **유대(Bond)**라는 서식으로 규칙화합니다. 유대는 서사적 장치이자 기계 효과이며, 특히 **커뮤플레이·장기 캠페인**에서 PC의 지속성을 보장합니다.

## 9-A. 유대란 무엇인가

유대는 PC와 특정 인물(PC 또는 NPC) 사이의 **이름 있는 관계**입니다.

- **정의**: 한 줄 설명 + 카테고리 + 관계 수치 (−5 ~ +5)
- **슬롯**: PC당 **3개 표준** (시작 시). 성장으로 확장.
- **효과**: 서사적 + 기계적 양면. 판정 보너스, 자원 회수, 감정 타격 등.

유대는 캐릭터 시트의 독립 칸에 기록합니다.

## 9-B. 6 카테고리

| 카테고리 | 전형 | 예시 |
|---|---|---|
| **가족·혈연** | 피로 이어진 관계 | 부모·형제·자매·자녀 |
| **동료·의리** | 약속으로 묶인 관계 | 팀원·옛 동료·은사 |
| **연인·애정** | 사랑·애정으로 묶인 관계 | 연인·배우자·짝사랑 |
| **은인·빚** | 받거나 진 빚의 관계 | 생명의 은인·돈·의무 |
| **라이벌·복수** | 적대·경쟁 관계 | 원수·라이벌·숙적 |
| **스승·제자** | 가르침으로 묶인 관계 | 스승·사부·제자 |

PC가 가질 수 있는 유대 카테고리는 **제한 없음**. 세 슬롯 안에서 어떻게 섞어도 무방합니다. "연인·애정" 1개 + "가족·혈연" 2개일 수도, "라이벌·복수" 2개 + "은인·빚" 1개일 수도 있습니다.

## 9-C. 관계 수치 (−5 ~ +5)

관계의 깊이·질을 표현하는 수치. **음수도 가능**합니다.

| 수치 | 의미 |
|---|---|
| +5 | 목숨을 걸어도 좋은 관계 |
| +3 | 깊은 신뢰 |
| +1 | 따뜻한 호의 |
| 0 | 중립·멀어진 관계 |
| −1 | 가벼운 불쾌 |
| −3 | 증오 |
| −5 | 죽이고 싶은 관계 |

**같은 상대에 대해 +5와 −5가 동시에 가능합니다**. "사랑한 만큼 증오하는" 관계는 PC 시트에 **+3 / −3** 같은 복합 수치로 기재할 수 있습니다(GM 재량).

## 9-D. 유대 효과 — 판정 보너스

유대 상대와 관련된 판정에 보너스가 붙습니다.

### 긍정 관계 (+1 이상)

- **동료 구하기**: 유대 상대를 구하려는 판정에 **+1 Die**.
- **약속 지키기**: 유대 상대와의 약속을 지키는 판정에 **+1 Die**.
- **관계 강화**: 관계 깊이 +1 할 때마다 Luck 토큰 1 회수 (세션 1회 한정).

### 부정 관계 (−1 이하)

- **복수 판정**: 유대 상대에게 해를 입히는 판정에 **+1 Die**.
- **대립 판정**: 유대 상대와의 대립 판정에 **에고 저항 +1**.
- **증오 계속 관계 유지**: 부정 관계 해소를 피하면 Luck 토큰 1 회수 (세션 1회).

긍정·부정 모두 **기계 효과가 있습니다**. "따뜻한 관계가 강하고 적대 관계는 약하다"가 아닙니다. 양쪽 다 서사의 원료이므로 양쪽 다 강화됩니다.

## 9-E. 감정 타격 — 유대 상대의 위협

유대 상대가 위협받을 때 PC는 **에고 타격**을 받습니다.

- **유대 상대 부상**: 에고 저항 판정 (TN 10). 실패 시 에고 −1d6.
- **유대 상대 죽음**: 에고 저항 판정 (TN 15). 실패 시 에고 −2d6 + 트라우마 획득 리스크.
- **유대 상대 배신**: 에고 저항 판정 (TN 20). 실패 시 에고 −2d6 + 관계 수치 완전 변경.

이 판정은 **유대 수치 |X|** 만큼의 페널티를 가집니다. 깊은 관계(|5|)일수록 저항이 어렵습니다.

## 9-F. 유대 변화 이벤트

유대는 고정이 아닙니다. 세션 중 사건으로 **강화·약화·깨짐·회복**됩니다.

### 강화
- 함께 위기 극복
- 깊은 대화·비밀 공유
- 서로를 위한 희생
- → 관계 수치 +1

### 약화
- 갈등·오해
- 약속 파기
- 다른 관계 우선
- → 관계 수치 −1

### 깨짐
- 배신
- 되돌릴 수 없는 상처
- 죽음·상실
- → 유대 슬롯 비움 (새 유대 획득 가능)

### 회복
- 화해·사과
- 증명의 행위
- → 관계 수치 0으로 복귀

GM은 세션 중 유대 변화 트리거를 적극적으로 제시합니다. "지금 당신의 선택이 이 유대를 바꿉니다"의 순간이 FEWK 플레이의 긴장점이 됩니다.

## 9-G. PC 간 유대

같은 팀의 PC끼리 유대를 맺는 것은 **강력히 권장**됩니다. 세션 0에서 최소 하나씩.

```
PC A ↔ PC B 유대 설정 예:
- 카테고리: 동료·의리
- 한 줄: "3년 전 동쟁 때 빅터의 손에 죽을 뻔한 걸 B가 구해줬다"
- 수치: A → B +3, B → A +1
```

PC 간 유대는 **일방향일 수 있습니다**. A가 B를 더 깊이 신뢰할 수도, B가 A를 더 좋아할 수도.

**금지 사항**: 다른 PC의 유대 수치를 그 플레이어의 동의 없이 바꿀 수 없습니다. "내 PC는 너를 증오한다"는 선언은 상대 플레이어의 허락이 필요합니다.

## 9-H. 커뮤플레이에서의 유대

커뮤플레이(13부 §04) 환경에서 유대는 **파티 간 연결 장치**로 확장됩니다.

- 다른 파티의 PC를 유대 상대로 설정 가능
- 세션 간 정보가 유대를 통해 전달
- 유대 상대 PC의 대아크 사건이 본 파티에 영향

커뮤플레이 진행자는 유대 그래프를 공용 도구(위키·노션)로 시각화합니다.

## 9-I. 유대 시트 템플릿

```
┌─ 유대 슬롯 1 ────────────────┐
│ 상대: ___________________    │
│ 카테고리: ________________    │
│ 한 줄:                       │
│ ___________________________  │
│ 수치: 본인→상대 [ ] 상대→본인 [ ] │
└─────────────────────────────┘

┌─ 유대 슬롯 2 ────────────────┐
...
```

부록 A 캐릭터 시트에 3 슬롯 유대 칸이 포함됩니다.

## 9-J. 예시 — 민의 유대 구성

> **민의 유대 3 슬롯**:
>
> **1. 가족·혈연 — 여동생 하얀** (민 → 하얀 +5)
> "하얀이 아프다. 내 모든 러닝의 이유."
>
> **2. 동료·의리 — 브로커 페이** (민 → 페이 +2, 페이 → 민 +3)
> "페이가 첫 의뢰를 주었다. 그 이후로 계속 일한다."
>
> **3. 라이벌·복수 — 갱단 두목 빅터** (민 → 빅터 −4)
> "빅터는 내 옛 동료를 죽였다. 아직 복수를 못했다."

이 세 유대가 민의 모든 결정을 설명합니다. 하얀의 병이 깊어지면 민은 보수를 올린다. 페이가 위기에 처하면 민은 무료로 돕는다. 빅터를 만나면 민은 침착함을 잃는다.
