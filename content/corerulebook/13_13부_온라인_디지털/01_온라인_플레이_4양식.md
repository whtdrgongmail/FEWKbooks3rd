# 13-1. 온라인 플레이 4 양식 개관

---

FEWK는 오프라인 테이블뿐 아니라 **네 가지 온라인 양식**으로 구현됩니다. 각 양식은 다른 시간 흐름·다른 인원 규모·다른 서술 밀도를 가지며, 캠페인 성격에 맞는 선택이 중요합니다.

## 1-A. 네 양식 개요

| 양식 | 동기성 | 시간 흐름 | 플랫폼 | 인원 |
|---|---|---|---|---|
| **ORPG** | 동기 (실시간) | 3~5시간/세션 | Discord Voice·Zoom·IRC | 3~6명 |
| **커뮤플레이** | 혼합 | 주 단위 | 공유 위키·노션·Discord | 10+ (다중 파티) |
| **PBeM** | 비동기 | 일·주 단위 턴 | 이메일 | 3~6명 |
| **게시판 플레이** | 비동기 | 게시글 단위 | 웹 포럼·Discord 포럼 | 3~6명 |

## 1-B. ORPG — 실시간 온라인 플레이

**성격**: 오프라인 세션의 온라인 변환. 음성 채팅·화면 공유.

**장점**: 실시간 상호작용, 오프라인과 가장 유사, 감정 전달 즉시.
**단점**: 시간대 조율, 네트워크 장애, 참여자 전원 동시 출석.
**적합 캠페인**: 전통 TRPG 경험·집중형 단편.

## 1-C. 커뮤플레이 — 공유 세계 운영

**성격**: 여러 파티·여러 GM이 **같은 세계관·타임라인 공유**. 한국 TRPG 커뮤 문화 특유.

**장점**: 세계관 풍부화, 다중 파티 상호작용, 장기 서사 확장.
**단점**: 진행자(Community Manager) 역할 필요, 복잡한 조율, 개인 플레이 리듬 손상 가능.
**적합 캠페인**: 한국 디스코드 커뮤·카페·길드 기반.

## 1-D. PBeM — 이메일 릴레이

**성격**: 이메일 스레드로 비동기 진행. 1~7일 단위 턴.

**장점**: 시간대 자유, 깊이 있는 서사 묘사, 기록 자동 보존.
**단점**: 느린 리듬, 이탈 위험 (응답 지연), 전투 처리 복잡.
**적합 캠페인**: 긴 서사·적은 전투·사회극 중심.

## 1-E. 플레이 바이 게시판 — 웹 포럼 턴제

**성격**: 포럼·게시판 스레드에 글 올려 진행. 공개·아카이브·관전자 존재.

**장점**: 공개 아카이브, 관전자 참여, 턴 구조 명확.
**단점**: 매우 느림 (게시글 단위), 공개로 인한 사생활 이슈, 포럼 관리 필요.
**적합 캠페인**: 커뮤 기반·공개 캠페인·아카이브 중시.

## 1-F. 양식 선택 플로우차트

```
「실시간 동시 출석 가능한가?」
    │
   Yes → ORPG
   No  → 「여러 파티가 세계관 공유하는가?」
             │
            Yes → 커뮤플레이
            No  → 「응답 주기 얼마?」
                      │
                     일~주 단위 → PBeM
                     게시글 단위 → 게시판 플레이
```

## 1-G. 하이브리드 플레이

둘 이상의 양식을 **결합** 가능:

### ORPG + 게시판 로그
- 실시간 세션 + 공개 아카이브
- 세션 녹화 → 게시판 요약

### 커뮤플레이 내 ORPG·PBeM 병용
- 각 파티가 선호 양식 선택
- 진행자가 통합 조율

### PBeM + 결정적 순간 ORPG
- 일상은 PBeM
- 전투·대결정은 실시간 ORPG

## 1-H. 양식 전환

캠페인 중 **양식 전환** 가능. 세션 0 재소집 → 양식 재합의 → 기존 PC 유지.

## 1-I. 양식별 판정·자원 처리

| 자원 | ORPG | 커뮤 | PBeM | 게시판 |
|---|---|---|---|---|
| 주사위 굴림 | 봇·실시간 | 봇·공유 | 봇·이메일 | 전용 스레드 |
| 판정 속도 | 즉시 | 즉시~일 | 일 단위 | 게시글 단위 |
| 운 토큰 | 실시간 선언 | 서면 선언 | 이메일 명시 | 게시글 명시 |
| DSR | 실시간 | 공유 문서 | 이메일 | 스레드 기록 |
