# 13-7. 비동기 플레이 공통 원칙

---

PBeM (§05), 게시판 플레이 (§06), 커뮤플레이 (§04) 세 양식은 모두 **비동기** 특성을 공유합니다. 이 장은 세 양식에 공통 적용되는 규약을 정리합니다.

## 7-A. IC/OOC 분리 원칙

### IC (In-Character)

- PC의 말·행동·생각
- 세계 내부 서술
- "민은 문을 두드린다."

### OOC (Out-of-Character)

- 플레이어의 질문·잡담·메타 협의
- 게임 외부 메모
- "죄송, 판정 TN이 몇이었죠?"

### 구분 표기 규약

- **이탤릭**: IC (글자 스타일)
- **대괄호**: `[OOC]` or `[IC]`
- **접힌 글**: OOC는 접힌 형태로
- **별도 스레드**: OOC 전용 채널

### 섞임 방지

- **한 글 안에서도 명확히** 구분
- IC 중 OOC 삽입 최소화
- OOC는 가능하면 별도 단락

## 7-B. 응답 주기 약속

### 세션 0에서 합의

```
양식별 권장 주기:
- PBeM: 3일~1주
- 게시판: 하루 1회 ~ 3일 1회
- 커뮤플레이: 주 단위
```

### 지연 시 알림

- **24시간 초과**: 간단 OOC 공지
- **3일 초과**: 상세 공지 (이유·예상 복귀)
- **1주 초과**: PC 처리 결정 (NPC 전환·이탈)

### GM 턴 주기·플레이어 턴 주기 분리

```
GM 턴: 플레이어 응답 수집 → 통합 → 새 서술 (3~7일)
플레이어 턴: GM 서술 수령 → 응답 (1~3일)
```

## 7-C. 드롭아웃 관리

### 단기 부재 (1~2주)

- PC 배경 이동 서술 (합리적 이유)
- 조연 NPC로 임시 대체
- 복귀 시 원상태

### 장기 부재 (1달+)

- 세 가지 선택:
  - **A**: PC 은퇴 (서사적 마무리)
  - **B**: NPC 전환 (GM 운영)
  - **C**: 파티 이탈 (관련 유대 해제)

### 침묵 이탈 방지 프로토콜

```
1주 무응답 → OOC 연락 (GM)
2주 무응답 → 파티 OOC 회의
3주 무응답 → 이탈 결정
1달 무응답 → PC NPC 처리
```

## 7-D. 판정 투명성

### GM 공개 굴림 vs 비공개

- **공개**: 모든 판정 봇 or 로그 첨부 → 신뢰 ↑
- **비공개**: GM 믿음 기반 → 신뢰 약화 시 분쟁

### 결과만 보고 (부분 투명)

- 판정 대상·TN은 공개
- 구체 굴림은 GM 비공개
- 결과만 본문에 기록

### 결과 조작 방지

- 주사위 봇 로그 (Discord·Roll20)
- 스크린샷 아카이브
- 분쟁 시 로그 공개

## 7-E. 서술 분량 균형

### 발언권 분배

한 사람이 독점하면 리듬 무너짐:
- GM 응답: 플레이어 발언 = 약 1 : 1~2
- 한 PC가 매 턴 길게 쓰면 다른 PC 부담

### "쉬는 턴"

- 판정 없이 묘사만
- 이동·관찰·감상
- 한 번씩 필요 (페이스 조절)

### 다른 PC 서술 참여

- 다른 PC의 행동 강요 금지
- "나는 [이렇게 보인다]"만 가능
- 다른 PC의 감정·선택은 그 플레이어의 것

## 7-F. 아카이브·데이터 보존

### 자동 수집

- **이메일 스레드** (PBeM)
- **게시판 스냅샷** (게시판)
- **Discord 로그** (커뮤플레이)

### 백업 주기

- **주 1회** 권장
- 백업 파일 2곳 이상 (클라우드 + 로컬)

### 요약본 작성

- **월 1회** 또는 아크 종료 시 **대요약**
- 10~30 세션을 1~3 페이지로 압축
- 다음 세션 시작 시 복귀 보조

### 플랫폼 이전

- 이메일 → 게시판 이전 시 포맷 변환
- 게시판 → Discord 이전 시 스레드 구조 재구성
- 장기 아카이브는 **markdown or txt** 형식 권장

## 7-G. 비동기 플레이 특수 이슈

### 판정 해석 충돌

- 비동기이므로 실시간 토론 불가
- OOC 쓰레드에서 48시간 논의
- 결정 못 나면 GM 최종 (또는 투표)

### PC 간 상호작용 지연

- PC A 행동 → PC B 반응까지 3일
- 긴 대화는 GM이 요약
- 중요 대화는 실시간 ORPG로 전환 가능

### 페이스 상실

- 너무 느리면 흥미 저하
- 2주 이상 무응답 시 "페이스 재시작" 검토
- 필요 시 양식 전환 (ORPG로)
