# 10-2. 세션 5단계 구조

---

TRPG 세션은 무작정 흘러가지 않습니다. **5단계 구조**로 리듬을 타면 플레이어의 집중과 감정이 유지됩니다.

## 2-A. 5 단계

```
① 도입 (Opening)     — 상황 설정, 세션 목표 제시
② 전개 (Development) — 정보 수집, 판정, 조우
③ 전환 (Turning Point) — 예상치 못한 변수, 결정적 선택
④ 절정 (Climax)      — 주요 충돌, 핵심 판정
⑤ 수습 (Resolution)   — 결과 정리, 다음 세션 훅
```

## 2-B. 시간 배분 (3시간 기준)

| 단계 | 분량 | 비율 |
|---|---|---|
| 도입 | 30분 | 17% |
| 전개 | 60분 | 33% |
| 전환 | 30분 | 17% |
| 절정 | 45분 | 25% |
| 수습 | 15분 | 8% |

세션 길이에 따라 비율 유지 (2시간·4시간 모두 동일 분포).

## 2-C. 단계별 GM 체크리스트

### ① 도입 (30분)

```
[ ] 지난 세션 요약 (2~3분)
[ ] 현재 상황 설정
[ ] 세션 목표 1~2개 선언 (or 플레이어 제안)
[ ] 모든 PC 첫 장면 등장
[ ] 운 토큰 리셋 확인
```

### ② 전개 (60분)

```
[ ] 핵심 판정 2~4회
[ ] 정보·단서 제공
[ ] NPC 조우 2~3명
[ ] 플레이어 창의적 해결 수용
[ ] Position 악화 징후 제시 (다음 단계 예고)
```

### ③ 전환 (30분)

```
[ ] 예상치 못한 변수 도입
[ ] Position 변화 (Risky → Desperate 등)
[ ] 플레이어 결정적 선택 강제
[ ] 감정적 무게 고조
```

### ④ 절정 (45분)

```
[ ] 핵심 판정·전투
[ ] 모든 PC 활약 기회
[ ] Critical·Fumble 극적 순간
[ ] Position Desperate 가능
[ ] 드라마 최고조
```

### ⑤ 수습 (15분)

```
[ ] 결과 서술
[ ] 애프터매스 6단계 (§09 전투 종료)
[ ] 자원 상태 확인
[ ] 다음 세션 훅 제시
[ ] 이정표·포인트 지급
[ ] Check-in (감정 확인)
```

## 2-D. 페이스 조정

### 너무 빠를 때

- 판정 간 서술 시간 늘림
- NPC 대화 확장
- 환경 묘사 추가

### 너무 느릴 때

- 판정 간 "진행" 선언
- "시간 건너뛰기" (주도 GM)
- 핵심 장면으로 직행

### 플레이어 집중 저하

- Check-in 요청
- 쉬는 시간 제안 (5~10분)
- 다음 판정 긴장도 높임

## 2-E. 즉흥 전환

세션 계획이 어긋날 때:
- **플레이어가 예상 외 경로 선택**: §12 즉석 구역 생성 테이블 활용
- **NPC 사망·도주**: 대체 NPC 등장 (§4 NPC 운영 참조)
- **시간 부족**: 절정을 다음 세션으로 (Clock 장치)

## 2-F. 한 세션의 착지

수습 단계는 **다음 세션 준비**이기도 합니다.

- "다음 세션에 당신들은 ___를 마주하게 됩니다"
- 플레이어가 기대감 갖도록
- 너무 구체적이지 않게 (자유도 유지)

## 2-G. 예시 — 한 세션 구조

> **세션 제목: "흑공의 저택"**
>
> **도입 (30분)**: 브로커 페이 등장. 실종된 동생 탐색 의뢰. 퓨어로열 지도 공유.
>
> **전개 (60분)**: 저택 접근. 주변 목격자 3명 인터뷰. 저택이 하루 중 특정 시간에만 출현함 감지.
>
> **전환 (30분)**: 저택 출현. PC들 진입 결정. 흑공 농도 5 진입. 매팅 저항 판정 시작.
>
> **절정 (45분)**: 저택 심부에서 먹개비 1등급과 조우. 전투. 실종된 동생 발견 (부분 매팅).
>
> **수습 (15분)**: 귀환. 페이에게 결과 보고. 매팅된 동생 치료 시도 시작. 다음 세션 훅: "저택이 왜 나타났는가."
