# 10-6. 문제 상황 대응·중단·저장

---

아무리 준비해도 세션은 예상 밖의 순간을 만납니다. 규칙 해석 충돌, 플레이어 간 갈등, 감정 사건, 시간 초과. GM의 **문제 대응 능력**이 테이블 신뢰를 지킵니다.

## 6-A. 5 유형 문제

| 유형 | 대응 |
|---|---|
| 규칙 해석 충돌 | 잠시 중단 → 규칙서 확인 → 결정 |
| 플레이어 간 감정 사건 | 안전 툴 호출 → 개인 대화 |
| GM 판단 뒤집기 | 플레이어 의견 수용 → 규칙 재확인 |
| 세션 중단 필요 | 조기 종료 절차 |
| 시간 초과 | 세션 저장 → 다음 회 이어감 |

## 6-B. 규칙 해석 충돌

### 대응 프로토콜

1. **잠시 중단**: "잠깐 규칙 확인할게요."
2. **규칙서 찾기**: 관련 페이지 확인 (또는 치트 시트)
3. **결정 선언**: "규칙상 X입니다. 적용하겠습니다."
4. **이견 수용**: 플레이어가 이의 제기하면 한 번 더 논의 후 **GM 최종**.

### 원칙

- **플레이어 유리 원칙**: 모호한 상황에서 플레이어 편
- **빠른 결정**: 세션 리듬 유지 (2분 이내)
- **사후 재검토**: 세션 후 규칙 재확인, 다음 세션 수정

## 6-C. 플레이어 간 감정 사건

### 대응 프로토콜

1. **즉시 중단**: "잠깐 쉬겠어요."
2. **안전 툴 확인**: X-Card 발동 여부
3. **개별 대화**: 당사자들과 개인 대화 (세션 외)
4. **복귀 or 종료**: 합의 후 계속 or 세션 조기 종료

### GM 중립

- 한 쪽 편 들지 않음
- 양쪽 의견 경청
- 타협점 제안

## 6-D. GM 판단 뒤집기

GM도 실수합니다. 플레이어가 더 잘 아는 규칙·상황이 있을 때:

- **인정**: "맞아요, 제가 잘못 판단했습니다."
- **수정**: 해당 판정·결과 재조정
- **사과 없음도 가능**: 인정만으로 충분

**GM 체면 유지 > 테이블 신뢰** 선택은 잘못된 것입니다. 실수 인정이 신뢰를 강화합니다.

## 6-E. 세션 중단·조기 종료

### 중단 이유

- 플레이어 감정 사건
- 기술 문제 (온라인)
- 시간 부족
- 예상 밖 사건 (현실 생활)

### 종료 절차

1. **선언**: "오늘 세션은 여기까지 하겠습니다."
2. **간략 애프터매스**: 자원 상태 기록
3. **상태 저장**: §11 세션 로그 참조
4. **다음 세션 예고**: 다음 시작점

## 6-F. 세션 저장

### 저장 대상

- PC 상태 (HP·에고·DSR·Pool·트라우마·트랙·히트)
- 현재 위치·시간
- 진행 중인 판정·Clock
- 중요 NPC 상태
- 다음 시작점 메모

### 저장 형식

GM 노트·공유 문서·VTT 로그 중 선택. 상세는 §11.

## 6-G. 세션 복귀

### 다음 세션 재개

```
재개 프로토콜:
1. 지난 세션 요약 (2~3분)
2. PC 상태 확인
3. 현재 상황 재설정
4. 플레이어 질문·확인
5. 계속 진행
```

## 6-H. 문제 유형 대응 치트 시트

```
[문제 발생]
    │
[유형 파악]
    │
규칙 충돌 ──→ 규칙 확인 ──→ 결정 ──→ 계속
    │
감정 사건 ──→ 중단 ──→ 개인 대화 ──→ 복귀/종료
    │
시간 초과 ──→ 세션 저장 ──→ 다음 회
    │
기술 문제 ──→ 30분 시도 ──→ 안 되면 종료
```

## 6-I. 예시 — 규칙 충돌

> **상황**: 플레이어 A가 "운 토큰으로 Position 완화 가능한가?" 질문.
>
> **GM 대응**:
> - "잠깐 확인할게요." (5부 §02 참조)
> - "Core §7-2-D 분담 원칙: Position은 장르 자원 전담. 운 토큰은 불가입니다."
> - "DSR 1 쓰시면 됩니다. 현재 DSR 7이니 여유 있네요."
> - **플레이어 수용**: "알겠어요, DSR 쓸게요."

2분 내 해결.
