# 03-6. 수정치 합산 순서·펌블 심화

---

판정값은 여러 수정치가 겹쳐 만들어집니다. 모두 더하면 되지만, **더하는 순서**가 분쟁이 생길 때 기준이 됩니다. 이 장은 그 순서와, 펌블·대실패 처리를 정리합니다.

## 6-A. 수정치 합산 순서

```
판정값 = 
  ① 스탯 Grade (굴림 + 폭발 체인)
+ ② 스킬 플랫
+ ③ 태그 수정치 (육체·성격·배경)
+ ④ 환경 수정치 (조명·거리·흑공 농도·엄폐 등)
+ ⑤ 임플란트·장비 수정치 (Pack)
+ ⑥ 운 토큰 추가 주사위 (선택: +스탯 Tier 주사위 1개)
+ ⑦ 상황 수정치 (GM 재량, 최후 적용)
```

### 왜 순서가 중요한가

대부분의 판정은 순서에 상관없이 결과가 같습니다(덧셈은 교환법칙이 성립). 그러나 다음 경우 순서가 영향을 미칩니다:

- 상한·하한 체크 (예: 스킬 상한 7 검증)
- 조건부 수정치 (Pack 특수 조건으로 "스탯 Grade가 적용된 뒤 +N")
- 분쟁 발생 시 기준 (플레이어와 GM의 해석 차이)

### 수정치 권장 범위

| 종류 | 범위 |
|---|---|
| 스킬 | +0 ~ +7 |
| 태그 | +0 ~ +3 |
| 장비 (Pack) | +0 ~ +5 |
| 환경 (단일 요소) | −10 ~ +5 |
| 운 토큰 보강 | +1 Tier Die (평균 2.5~10.5) |

극한 누적 시 GM은 "이성적 한계"를 기준으로 판단합니다.

### 부정 수정치 상한

```
상황 페널티 누적이 −15 이하 → 판정 불가 (자동 실패)
극한 환경 페널티 −20+ → 판정 시도 불가, Pack 이벤트로 처리
```

예: 차원 붕괴 구역에서 정상 판정은 불가능합니다. GM은 Pack의 특수 이벤트(드랍아웃·진사 등)로 대체합니다.

## 6-B. Fumble — 모든 주사위가 1인 상태

### 정의

Grade Die에서 **모든 주사위가 1**을 내는 현상. 차이값과 **무관**하게 최우선으로 적용됩니다.

```
Fumble 확률 (개수 N, 면수 F일 때):
Fumble = (1/F)^N

1d4: 25%
4d4: 0.39%
3d6: 0.46%
5d6: 0.013%
6d8: 0.00004%
10d10+: 사실상 0
```

### 효과

| 판정 유형 | Fumble 효과 |
|---|---|
| 공격 | 무기 장애 (1d6 라운드 사용 불가) 또는 동료 피해 50% 리스크 |
| 스킬 | Position 1단계 악화 + 자동 실패 |
| 저항 (Grade Die) | 자동 실패 + 피해 1 또는 상태 이상 |
| d100 (펌블 00) | 자동 실패 + Pack 특수 이벤트 |
| d6 Pool (모두 1) | 연료 낭비 + 역효과 (Pack 지정) |

### Fumble 회피 — 운 토큰 2

```
운 토큰 2 소모 → 펌블 자동 재굴림
재굴림 결과가 또 Fumble → 효과 반감 (GM 판정)
```

Fumble 회피는 고위험 순간의 안전 장치입니다. 운 토큰 상세는 5부 §02.

## 6-C. Fumble vs 대실패 — 용어 정리

두 개념은 자주 혼동됩니다. **완전히 다른 현상**입니다.

| 구분 | Fumble | 대실패 |
|---|---|---|
| 원인 | 주사위 굴림 자체 (모든 1) | 판정값 − TN = −10 이하 |
| 발동 시점 | 굴림 결과 즉시 | TN 비교 후 |
| 회피 방법 | 운 토큰 2 | 협상·재시도 (Position 의존) |
| 우선순위 | 최우선 | TN 비교 이후 |

Fumble과 대실패가 **동시에** 발생하는 경우, Fumble 효과만 적용합니다. "자동 실패 + 최악 대가"가 이미 포함된 결과이기 때문입니다.

> **예시 1 — Fumble + 대실패**:
> 2d4 (스탯 2) 굴림 → 1, 1 → Fumble.
> 판정값 2 + 스킬 0 = 2. TN 15와 차이 −13 → 대실패 범위.
> → **Fumble 효과만 적용**.

> **예시 2 — 대실패만**:
> 2d4 굴림 → 1, 3 → Fumble 아님.
> 판정값 4. TN 15와 차이 −11 → 대실패.
> → **대실패 효과 적용**.

## 6-D. d100 펌블 (00)

d100에서 **00(= 100)**이 나오면 펌블. Grade Die의 Fumble과는 별개의 현상입니다.

**전형적 d100 펌블**:
- 세계법칙 저항: Pack 특수 이벤트 발동 (매팅 +2·샤이닝 +2·패러독스 +1 등)
- 랜덤 테이블: 최악의 결과
- 커먼센스 확률: 완전히 불가능한 상태

d100 펌블은 **운 토큰 2**로 회피 가능. 재굴림합니다.

## 6-E. d6 Pool 펌블 (모두 1)

d6 Pool을 N개 소모했는데 **모두 1**이 나오는 경우. 확률은:

```
Pool 펌블 확률 = (1/6)^N

5d6 Pool: 0.013%
10d6 Pool: 0.0000002%
```

현실적으로 5d6 이상에서는 사실상 면역이지만, 발동 시 효과는 가혹합니다.

**효과**:
- 연료 낭비 (소모한 개수 모두 소실)
- 역효과 발동 (Pack 지정 — 기관포 폭발·신기 역류 등)
- 상태 이상 추가 가능성

## 6-F. 수정치 분쟁 해결 순서

판정 중 수정치 분쟁이 생겼을 때 GM은 다음 순서로 판단합니다.

```
1. 규칙 문서에 명시된 수정치 우선 (이 책·엔진 3종)
2. Pack 문서에 명시된 수정치 (FEWK Cyberpunk Pack)
3. GM 재량 수정치 (§6-6-A ⑦)
4. 분쟁 시 플레이어 유리로 (관례)
```

네 번째 원칙 — "분쟁 시 플레이어 유리" — 는 FEWK 3판의 테이블 문화 원칙입니다. GM은 모호한 상황에서 플레이어 편을 드는 것을 기본값으로 합니다. 이는 테이블 신뢰를 보장합니다.

> 🎯 **GM 팁** — Fumble과 대실패를 혼동하는 분쟁이 가장 흔합니다. 굴림 직후 한 박자 멈춰 "이건 모든 주사위가 1인가? 아니면 차이값이 −10 이하인가?"를 머릿속으로 순서대로 체크하십시오. 둘이 동시에 떨어졌을 때는 Fumble 하나만 적용 — 두 번 처벌하지 않습니다.

## 6-G. 수정치 체크리스트

판정 직전 GM이 빠르게 훑는 체크리스트:

```
[ ] 스탯 Grade 확인
[ ] 관련 스킬 수치 확인 (상한 검증)
[ ] 태그 수정치 적용 (해당 시)
[ ] 환경 수정치 (조명·거리·농도 등)
[ ] 장비·임플란트 수정치
[ ] 운 토큰 추가 선언 확인
[ ] 상황 수정치 (즉흥)
[ ] Position·Effect 선언
[ ] TN 선언
```

체크리스트는 부록 B 치트 시트에 재수록됩니다.
